# CLAUDE.md — OpenClaw Telegram AI Agent Gateway

이 파일은 Claude Code가 이 프로젝트에서 작업할 때 항상 숙지해야 할 내용이다.

---

## 프로젝트 개요

`openclaw` NPM 패키지를 활용하여 Telegram 봇(`@openclaw_claude_da_bot`)과 Google Gemini API를 연결하는 AI 에이전트 게이트웨이. gcube 컨테이너 환경에서 `git clone → bash setup.sh → bash run.sh` 3단계로 배포된다.

**앱 로직은 이 repo에 없다.** 실제 게이트웨이, 채널 핸들러, 봇 통신은 모두 `openclaw` NPM 패키지가 담당한다. 이 repo는 설정 파일을 생성하는 셸 스크립트와, 에이전트 동작을 정의하는 bootstrap 파일(`identity/`)만 포함한다.

---

## 핵심 원칙 (절대 위반 금지)

1. **Claude 관련 코드를 절대 추가하지 않는다.** `ANTHROPIC_API_KEY`, `anthropic/claude-*` 모델 ID, Claude 분기 로직은 어떤 파일에도 포함해서는 안 된다.
2. **AI 백엔드는 `google/gemini-2.5-flash` 단일 모델만 사용한다.** 모델 선택 분기, 변수화, 다른 모델 추가는 금지.
3. **WebUI(브라우저 Control UI)는 사용 가능하다.** `controlUi` 설정을 포함하며, Gateway Token 안내 및 브라우저 접속 흐름을 지원한다.
4. **현재는 Telegram 단독 운영이며, 추후 Slack을 추가할 예정이다.** 현재 `openclaw.json`의 `channels`에는 telegram만 활성화되어 있다.
5. **파일 수를 최소화한다.** 소스 코드 디렉토리(`src/`, `lib/` 등)나 추가 설정 파일을 만들지 않는다. 필요한 변경은 기존 스크립트를 수정한다.

## 주요 파일 역할

| 파일 | 역할 | 수정 빈도 |
|------|------|----------|
| `setup.sh` | `.env` 읽기 → openclaw 설치 → 설정 파일 생성 → identity/docs 복사 | 낮음 |
| `run.sh` | 설정 검증 → 게이트웨이 백그라운드 실행 → 페어링 안내 출력 | 낮음 |
| `.env.example` | `GOOGLE_API_KEY`, `TELEGRAM_BOT_TOKEN` 키 템플릿 | 거의 없음 |
| `identity/AGENTS.md` | 에이전트 지시사항 (한국어 답변 설정) | 지시사항 변경 시 |
| `identity/IDENTITY.md` | 에이전트 정체성 (DA Assistant, 담당자님 호칭) | 정체성 변경 시 |
| `docs/*.md` | Knowledge Base 문서 (에이전트 참조용) | 문서 추가/변경 시 |
| `README.md` | Telegram 전용 배포 가이드 | 명세 변경 시 |

## `openclaw.json` 핵심 설정

`setup.sh`가 생성하는 `~/.openclaw/openclaw.json`의 고정 구조:

```json
{
  "agents": {
    "defaults": {
      "model": "google/gemini-2.5-flash"
    }
  },
  "gateway": {
    "port": 8080,
    "mode": "local",
    "bind": "lan",
    "auth": { "mode": "token", "token": "<auto-generated>" }
  },
  "channels": {
    "telegram": { "enabled": true, "botToken": "<from .env>", "dmPolicy": "pairing" }
  },
  "plugins": {
    "entries": { "telegram": { "enabled": true } }
  }
}
```

- `gateway.controlUi` 키를 포함한다 (WebUI 사용 가능, controlUi 설정 포함).
- 현재 `channels`에는 `telegram`만 활성화되어 있으며, 추후 Slack 채널 추가 예정이다.

## Workspace Bootstrap 시스템

에이전트 시스템 프롬프트는 `openclaw.json`이 아닌 **workspace bootstrap 파일**로 관리한다.

- `identity/AGENTS.md` → `~/.openclaw/workspace/AGENTS.md` (에이전트 지시사항)
- `identity/IDENTITY.md` → `~/.openclaw/workspace/IDENTITY.md` (에이전트 정체성)

이 파일들은 OpenClaw 에이전트 실행 시 자동으로 컨텍스트에 주입된다.

**에이전트 정체성 설정**:

| 항목 | 값 |
|------|-----|
| 이름 | DA Assistant |
| 역할 | 문서 생성, 편집, 분석 담당 어시스턴트 |
| 사용자 호칭 | 담당자님 |
| 톤 | 사무적이고 간결하게, 이모지 없이 |
| 언어 | 한국어 (모든 응답) |

---

## 작업 시 주의사항

- **모든 작업 시**: 응답은 한국어로 작성할 것.
- **`setup.sh` 수정 시**: `ANTHROPIC_API_KEY`, `MODEL`/`PROVIDER`/`API_KEY` 변수가 다시 생기지 않도록 한다. API 키 검증은 `GOOGLE_API_KEY` 단독으로만 한다.
- **`run.sh` 수정 시**: 출력 메시지에 WebUI 토큰 안내("Gateway Token", "gcube URL로 접속")와 Telegram 페어링 안내를 모두 포함한다.
- **`.env.example` 수정 시**: `ANTHROPIC_API_KEY`는 어떤 형태로도 추가하지 않는다.
- **`README.md` 수정 시**: Telegram 봇 이름은 `@openclaw_claude_da_bot`이다. Anthropic 발급 링크를 포함하지 않는다.
- **설정 JSON 수정 시**: 모델 ID는 반드시 `"google/gemini-2.5-flash"`이다.
- **`identity/` 수정 시**: 수정 후 `bash setup.sh`를 재실행해야 workspace에 반영된다.
- **`docs/` 수정 시**: 문서 추가/변경 후 `bash setup.sh`를 재실행해야 workspace에 반영된다.
- **스크립트는 `set -e`로 실행된다.** 오류 발생 시 즉시 종료되므로, 조건부 명령(`|| true` 등)을 적절히 사용한다.

## 자주 쓰는 명령어

```bash
# 설정 초기화 (최초 1회 또는 설정 리셋)
bash setup.sh

# 게이트웨이 실행
bash run.sh

# 로그 확인
cat ~/.openclaw/gateway.log

# Telegram 페어링 승인
openclaw pairing approve telegram [코드]

# 페어링 목록 확인
openclaw pairing list

# 게이트웨이 프로세스 종료
pkill -9 -f "openclaw" 2>/dev/null || true

# 게이트웨이 재시작 (좀비 프로세스 정리 포함)
pkill -9 -f "openclaw" 2>/dev/null || true && sleep 2 && bash run.sh

# 설정 파일 확인
cat ~/.openclaw/openclaw.json

# workspace 파일 확인
ls ~/.openclaw/workspace/
```
